IoT com ESP32, PHP
e MySQL

Aprenda como um microcontrolador ESP32 se conecta a uma rede Wi-Fi, coleta dados de sensores e os envia automaticamente para um banco de dados MySQL via PHP — utilizando o protocolo HTTP GET sobre uma rede local com XAMPP.

Componentes do Projeto

Cada camada da solução e sua responsabilidade.

ESP32 – Microcontrolador

Microcontrolador de 32 bits com Wi-Fi e Bluetooth integrados. Neste projeto, conecta-se à rede local, lê os sensores e envia os dados ao servidor usando a biblioteca HTTPClient.h. O código roda na Arduino IDE.

Wi-Fi – Comunicação

O ESP32 usa a biblioteca WiFi.h para se conectar a um roteador padrão (2.4 GHz). Após conectar, obtém um endereço IP e consegue alcançar o servidor XAMPP que estiver na mesma rede local.

HTTP GET – Protocolo

Os dados dos sensores são embutidos diretamente na URL como parâmetros (query string). É o método mais simples de integração entre hardware e web — sem necessidade de JSON ou autenticação para fins didáticos.

XAMPP – Servidor Local

Pacote que instala Apache, MySQL e PHP em um único clique. O Apache serve os scripts PHP na porta 80. O MySQL armazena os dados. O XAMPP simula um servidor web completo no próprio computador do laboratório.

PHP – Back-end

Dois scripts PHP compõem o back-end: recebe_dados.php valida os parâmetros GET e executa o INSERT no banco; lista_dados.php faz o SELECT e exibe os registros em uma tabela Bootstrap 5. Padrão de código baseado no W3Schools.

MySQL – Banco de Dados

Sistema gerenciador de banco de dados relacional (SGBD). O banco banco_iot contém a tabela tb_dados_recebidos com campos para temperatura, umidade, local, mensagem, equipamento e data/hora do registro.

Estrutura do Banco de Dados

PHP – recebe_dados.php

Recebe, valida e insere os dados no banco. Padrão W3Schools php_mysql_insert.asp.

recebe_dados.php // Conexão com o banco (padrão W3Schools)
$conn = new mysqli($servername, $username, $password, $dbname);

// Validação dos parâmetros GET
if (!isset($_GET["temperatura"])) {
  die("ERRO: Parametros incompletos.");
}

// Sanitização com htmlspecialchars()
$temperatura = trim(htmlspecialchars($_GET["temperatura"]));

// INSERT no banco (padrão W3Schools)
$sql = "INSERT INTO tb_dados_recebidos
       (temperatura, humidade, local,
       mensagem, equipamento, dt_registro)
       VALUES ('$temperatura', ..., NOW())";

if ($conn->query($sql) === TRUE) {
  echo "Registro inserido com sucesso!";
}

$conn->close();
Validação com isset()

Verifica se todos os parâmetros obrigatórios chegaram na URL antes de tentar acessá-los.

Sanitização com htmlspecialchars()

Converte caracteres especiais em entidades HTML, protegendo contra injeção de código (XSS).

NOW() no INSERT

A data e hora do recebimento são gravadas automaticamente pelo MySQL, sem dependência do relógio do ESP32.

O Que Alterar no Código

Dois pontos no firmware do ESP32 precisam ser ajustados para cada laboratório.

1. Nome e Senha da Rede Wi-Fi

Localize esta linha no setup() e substitua pelos dados da rede do laboratório:

WiFi.begin("Sua REDE WiFi aqui", "Senha do WiFi aqui");
2. IP do Servidor XAMPP

Descubra o IP da máquina com o XAMPP (ipconfig no Windows) e substitua:

minhaURL.concat("192.168.33.12"); // ← altere aqui
Passo a Passo para Executar

Siga a ordem abaixo para colocar o projeto em funcionamento.

1
Instalar e iniciar o XAMPP

Baixe o XAMPP em apachefriends.org, instale e inicie os módulos Apache e MySQL no painel de controle.

2
Criar o banco de dados

Acesse localhost/phpmyadmin, clique em SQL, cole o conteúdo do arquivo banco_iot.sql e execute.

3
Copiar os arquivos PHP

Crie a pasta htdocs/AulaIoT/ dentro do diretório do XAMPP e copie recebe_dados.php, lista_dados.php e index.html para ela.

4
Descobrir o IP do servidor

No Windows, abra o Prompt de Comando e execute ipconfig. Anote o Endereço IPv4 da interface de rede Wi-Fi.

5
Configurar e gravar o firmware

Na Arduino IDE, altere o SSID, a senha e o IP do servidor no código esp32_http_get.cpp. Conecte o ESP32 via USB e grave o firmware.

6
Monitorar e visualizar os dados

Abra o Serial Monitor (115200 bps) para acompanhar os envios em tempo real. Acesse lista_dados.php no navegador para ver os registros na tabela.
